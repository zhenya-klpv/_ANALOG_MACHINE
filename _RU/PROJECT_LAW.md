# Закон проекта — Bilingual Mirror

> **Обязательно для всех.** Нарушение = технический долг. AI и люди следуют без исключений.

---

## Закон

**Каждый новый или существенно изменённый файл документации существует в двух языках:**

| Язык | Путь |
|------|------|
| **Русский** | `_RU/` — та же структура папок, то же имя файла |
| **Английский** | `_ENG/` — та же структура папок, то же имя файла |

### Правило 1 — RU → EN

Добавил или обновил файл **в `_RU/`** → **переведи и синхронизируй** в `_ENG/`.

### Правило 2 — EN → RU

Добавил или обновил файл **в `_ENG/`** → **переведи и синхронизируй** в `_RU/`.

### Правило 3 — Один коммит, два языка

Одно логическое изменение = **оба файла** обновлены в одной сессии. Не оставлять зеркало устаревшим.

### Правило 4 — Та же структура

```
_RU/brand/BRAND.md          ↔  _ENG/brand/BRAND.md
_RU/builds/e36-00/BUILD_BRIEF.md  ↔  _ENG/builds/e36-00/BUILD_BRIEF.md
```

**В корне проекта** — только `README.md`, `.gitignore`, `.cursor/`. **Не** класть `.md` документацию в корень.

---

## Что синхронизировать

| Да | Нет |
|----|-----|
| `.md` в `_RU/` ↔ `_ENG/` | `.gitignore`, `.cursor/` (только корень) |
| Чеклисты, briefs | `private/` — по согласованию |

**Изображения:** копировать в `_RU/...` и `_ENG/...`. `INDEX.md` — переводить.

---

## Процесс

1. Правишь файл → определи язык (`_RU` или `_ENG`)
2. Синхронизируй зеркало
3. Ссылки: внутри `_RU/` — пути в `_RU/`; внутри `_ENG/` — в `_ENG/`
4. Структура изменилась → `AI_CONTEXT.md` в **обоих** языках
